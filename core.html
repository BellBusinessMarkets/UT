Background Information on the core _UT

The core of the _UT library is based of jQuery's Core, with some unused functionality removed.

jQuery Core provides some very useful functions such as jQuery.extend, jQuery.each, jQuery.globalEval,etc... Instead of reinventing the wheel, we decided it would be advantageous to use the community approved & tested code from jQuery to fill the blanks required in the _UT library.

Utilities List of core utility functions available from the _UT namespace

These utility functions are available to streamline the development of Adapters, plugins, processors and mappings by providing some commonly used functions.

Try them out! You can test these functions live using Firebug by simply clicking F12. The _UT library has been pre-loaded
Function Description & Parameters
_UT.extend

Merge the contents of two or more objects together into the first object.

  1. target An object that will receive the new properties if additional objects are passed in or that will extend the jQuery namespace if it is the sole argument.
  2. object1 An object containing additional properties to merge in.
  3. objectN Additional objects containing properties to merge in.

  1. deep If true, the merge becomes recursive (aka. deep copy).
  2. target An object that will receive the new properties if additional objects are passed in or that will extend the jQuery namespace if it is the sole argument.
  3. object1 An object containing additional properties to merge in.
  4. objectN Additional objects containing properties to merge in.
@return {obj}

View jQuery.extend for more information.

Ex: 
			
var a = { a:1, c:1};
var b = { b:1, a:2};

_UT.extend(a,b); // ==> a = { a:2, b:1, c:1}
_UT.addExtensions

Adds utility functions to the _UT namespace. Static functions.

  1. obj object where key is name of function and value is the new function
Ex: 
			
_UT.addExtensions({
	add: function(a,b) {
		return a+b;
	},
	substract: function(a,b) {
		return a-b;
	}
});

var sum = _UT.add(5,3); // ==> 8
var diff = _UT.substract(5,3); // ==> 2
_UT.addPlugins

Adds plugins to all or a specific Adapter Instances.

  1. obj object where key is name of function and value is the new function
  2. type [optional] Type of Adapter. Ex: "omni" or "ga"
Ex: 
			
_UT.addPlugins({
		getSettings: function() {
			return this.settings;	// all _UT Adapter instance have property map
		}
	}
	,"ga" // limits to only gaAdapters / if not present, attaches function to all adapter instances
);

// assuming _ut_ga is a GA Adapter instance
console.debug(_ut_ga.getSettings());	// ==> prints the settings to console for instantiated GA adapter
_UT.each

A generic iterator function, which can be used to seamlessly iterate over both objects and arrays. Arrays and array-like objects with a length property (such as a function's arguments object) are iterated by numeric index, from 0 to length-1. Other objects are iterated via their named properties.

  1. collection The object or array to iterate over.
  2. callback(indexInArray, valueOfElement) The function that will be executed on every object.


View jQuery.each for more information.

Ex: 
			
var sum = 0;
_UT.each([1,2,5], function(){
	sum = sum + this;
});
// ==> sum = 8

var prod = 1;
_UT.each({a: 2, b: 3, c: 4}, function(key,val){
	if(key.search(/[ac]/gi) != -1) prod = prod * val;
});
// ==> prod = 8

_UT.copyArray

Function to create a deep copy of an Array.

  1. obj Array to copy.
@return {array}
Ex: 
			
var myArr = [1,2,3];
var newArr = _UT.copyArray(myArr); // ==> [1,2,3]
_UT.copyObject

Function to create a deep copy of an Object.

  1. obj Object to copy.
@return {obj}
Ex: 
	
var obj = {g:4};		
var myObj = {a:1,b:2,c:{d:3},f:obj};
var newObj = _UT.copyObject(myObj); // ==> {a:1,b:2,c:{d:3}}
obj.g = 5; // ==> g = {g:5}

myObj; 	// ==> {a:1,b:2,c:{d:3},f:{g:5}}
newObj; // ==> {a:1,b:2,c:{d:3},f:{g:4}}

_UT.isNumeric

Determines whether its argument is a number.

  1. value The value to be tested.
@return {boolean}

View jQuery.isNumeric for more information.

Ex: 
	
_UT.isNumeric("-10");  // ==> true
_UT.isNumeric(16);     // ==> true
_UT.isNumeric(0xFF);   // ==> true
_UT.isNumeric("0xFF"); // ==> true
_UT.isNumeric("8e5");  // ==> true (exponential notation string)
_UT.isNumeric(3.1415); // ==> true
_UT.isNumeric(+10);    // ==> true
_UT.isNumeric(0144);   // ==> true (octal integer literal)
_UT.isNumeric("");     // ==> false
_UT.isNumeric({});     // ==> false (empty object)
_UT.isNumeric(NaN);    // ==> false
_UT.isNumeric(null);   // ==> false
_UT.isNumeric(true);   // ==> false
_UT.isNumeric(Infinity); // ==> false
_UT.isNumeric(undefined); // ==> false
_UT.type

Determine the internal JavaScript [[Class]] of an object.

  1. obj Object to get the internal JavaScript [[Class]] of.
@return {string}

View jQuery.type for more information.

_UT.type() // ==> "undefined"
_UT.type(window.notDefined) // ==> "undefined"
_UT.type(null) // ==> "null"

_UT.type(true) // ==> "boolean"
_UT.type(3) // ==> "number"
_UT.type("test") // ==> "string"
_UT.type(function(){}) // ==> "function"
_UT.type([]) // ==> "array"
_UT.type(new Date()) // ==> "date"
_UT.type(/test/) // ==> "regexp"

_UT.inArray

Search for a specified value within an array and return its index (or -1 if not found).

  1. value The value to search for.
  2. array An array through which to search.
  3. fromIndex The index of the array at which to begin the search. The default is 0, which will search the whole array.
@return {number}

View jQuery.inArray for more information.

Ex: 
	
var myArr = [1,"is", "the","number"];

_UT.inArray(1,myArr);	// ==> true
_UT.inArray("is",myArr);	// ==> true
_UT.inArray("the",myArr);	// ==> true
_UT.inArray("loneliest",myArr);	// ==> false
_UT.inArray("number",myArr);	// ==> true

_UT.grep

Finds the elements of an array which satisfy a filter function. The original array is not affected.

  1. array The array to search through.
  2. function(elementOfArray, indexInArray) The function to process each item against. The first argument to the function is the item, and the second argument is the index. The function should return a Boolean value. this will be the global window object.
  3. invert If "invert" is false, or not provided, then the function returns an array consisting of all elements for which "callback" returns true. If "invert" is true, then the function returns an array consisting of all elements for which "callback" returns false.
@return {array}

View jQuery.grep for more information.

Ex: 
	
var arr = [ 1, 9, 3, 8, 6, 1, 5, 9, 4, 7, 3, 8, 6, 9, 1 ];

arr = _UT.grep(arr, function(n, i){
  return (n != 5 && i > 4);
});
// ==> arr = [1, 9, 4, 7, 3, 8, 6, 9, 1]

arr = _UT.grep(arr, function (a) { return a != 9; });
// ==> arr = [1, 4, 7, 3, 8, 6, 1]
_UT.map

Translate all items in an array or object to new array of items.

  1. array The Array to translate.
  2. callback(elementOfArray, indexInArray) The function to process each item against. The first argument to the function is the array item, the second argument is the index in array The function can return any value. Within the function, this refers to the global (window) object.
  1. arrayOrObject The Array or Object to translate.
  2. callback( value, indexOrKey ) The function to process each item against. The first argument to the function is the value; the second argument is the index or key of the array or object property. The function can return any value to add to the array. A returned array will be flattened into the resulting array. Within the function, this refers to the global (window) object.
@return {array}

View jQuery.map for more information.

Ex: 
	
var arr = [1,2,3]

arr = _UT.map(arr,function(val,i){return val*i});
// ==> arr = [0, 2, 6]

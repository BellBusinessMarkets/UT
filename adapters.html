How To Create an Adapter

Can't find an Adapter for your Service of choice? Not happy with the existing ones ? Good news ! Creating an Adapter is not that hard !

The _UT Adapter for your service serves two functions:

  1. Enables you to hook up a new Service to the _UT infrastructure
  2. Enables you to create an abstraction layer for the Service to simplify the deployment and coding required
To Simplify: we will be creating a new Adapter for a brand new Analytics Provider called xyz.

Skeleton

First things first, we need to name our Adapter. This name will be be used as the type property in our Adapter settings parameter. Let's name our adapter xyz (very orginal, I know !).

Now we need to prepend our Adapter name to the word Adapter when defining our new Adapter class. This new class must extend the _UT.Adapter Interface

We therefore create a new file called xyzAdapter.js

			
// creating global exception handler //==> good practice to capture errors
_UT.exceptionHandler = _UT.exceptionHandler || new _UT.Exception();

/**
 * UT xyz Adapter.
 * @extends _UT.Adapter
 */
_UT.xyzAdapter = _UT.Adapter.extend({


	// Adapter Code will go here


}); // end extend

Mandatory Properties

Now that we have our skeleton for the xyzAdapter Class, we need to implement mandatory properties and methods. The list of these properties and descriptions is found in the table below:

Property Description & Parameters
init

Constructor method for Adapter Object. Requires 3 parameters: settings,processor and map as described in _UT.Adapter.init

To correctly enable the functionality of the _UT.Adapter Class, it is necessary to call the parent's init method using the reserved keyword _super

this._super(settings, processor, map);

After this step, the settings object can be used to correctly configure the xyz solution and create helper methods and values.Ex:

// Load xyz Library
if(typeof _xyz == "undefined")
	(function() {
		var ga = document.createElement('script'); ga.type = 'text/javascript'; ga.async = true;
		ga.src = ('https:' == document.location.protocol ? 'https://ssl' : 'http://www') + '.xyz-analytics.com/ga.js';
		var s = document.getElementsByTagName('script')[0]; s.parentNode.insertBefore(ga, s);
	})();

_xyz.setAccount(settings.account);
_xyz.setInternalSearchQueryParameter(settings.internal_search_qp);
_xyz.setCampaignQueryParameter(settings.campaign_qp);

// always return true
return true;
fire

The fire takes care of calling the solutions pre-built tracking function. Requires 1 parameter: vars as described in _UT.Adapter.fire

To correctly enable the functionality of the _UT.Adapter Class, it is necessary to call the parent's init method using the reserved keyword _super. This will take the vars object, clean it , pass it through the processor and map functions in order to prepare the Solution object for firing.

this._super(vars);

After this step, the solutions javascript object is ready to be fired which will send the data to the servers. To effectively trigger the beacons, we need to call the solutions fire beacon function. Ex:

_xyz.trackPageview(); // firing xyz beacon
info

This property is not mandatory but useful in keeping track of Adapter versions.

The info Object should contain 3 properties:

  • version Version of the Adapter.
  • date Date of last update
  • author Author of Adapter
version: "1.2",
date:    "2012-05-12",
author: "DD"

Optional Properties

Our xyzAdapter is now ready to be used. However, since in the map functions we will need access to the _xyz object in its native (and maybe complex) form, we can create some more methods to simplify the interactions with that Object.

Here are examples of methods simplifying interactions with the (hypothetical)_xyz.

purchaseProduct: function(products) {
	for(var i= 0; i<products.length;i++) {
		var prod = products[i];
		_xyz.addProduct(prod.name, prod.quantity, prod.price);
	}
	_xyz.sendProducts('purchase');
}

Final xyzAdapter Code

Finally, our code is complete. When we put all this together, here is the result:

_UT.exceptionHandler = _UT.exceptionHandler || new _UT.Exception();

/**
 * UT xyz Adapter.
 * @extends _UT.Adapter
 */
_UT.xyzAdapter = _UT.Adapter.extend({

    // Adapter info
    info: {
        version: "1.1.1",
        date:    "2012-03-12",
        author: "DD"
    },  
    
    /**
     * XYZ Adapter
     * @constructor
     */
    init: function (settings, processor, map) {
        try {

            // Load xyz Library
            if(typeof _xyz == "undefined")
                (function() {
                    var ga = document.createElement('script'); ga.type = 'text/javascript'; ga.async = true;
                    ga.src = ('https:' == document.location.protocol ? 'https://ssl' : 'http://www') + '.xyz-analytics.com/ga.js';
                    var s = document.getElementsByTagName('script')[0]; s.parentNode.insertBefore(ga, s);
                })();
            
            _xyz.setAccount(settings.account);
            _xyz.setInternalSearchQueryParameter(settings.internal_search_qp);
            _xyz.setCampaignQueryParameter(settings.campaign_qp);

            // always return true
            return true;
        } catch (e) {
            _UT.exceptionHandler.push(e);
        }
    },

    /**
     * method to fire beacon for XYZ
     */
    fire: function (vars) {

        // calling the UT Adapter function to clean up
        this._super(vars);
        
        _xyz.trackPageview(); // firing xyz beacon
        
    },

    /**
     * method to send purchase info to XYZ 
     */    
    purchaseProduct: function(products) {
        for(var i= 0; i<products.length;i++) {
            var prod = products[i];
            _xyz.addProduct(prod.name, prod.quantity, prod.price);
        }
        _xyz.sendProducts('purchase');
    }

}); // end extend

gaAdapter Using the Google Analytics Adapter

Background

Following the previous instructions, an Adapter was created for Google Analytics type-named ga.

Getting started

In order to use the gaAdapter Class, we simply need to instantiate a new object and then call the fire function with our specific variables

// Create Adapter Instance
_ut_ga = new _UT.gaAdapter(GA_SETTINGS, GA_MAP, GA_PROCESSOR);

// Fire Adapter beacon
_ut_ga.fire({
    __usePrevious: false,
    __useWindow: true
});

Evidently GA_SETTINGS, GA_MAP amd GA_PROCESSOR need to be defined already

Settings

Lets build the GA_SETTINGS object.

Property Table

In addition to the mandatory properties of the settings object as described in _UT.Adapter.settings, the Google Analytics settings objects requires many specific properties to work as expected:

Property Description & Parameters
id

The id will also be used to create the instance globally in Container

id: "_ut_ga", // same value as variable name _ut_ga
type

This property is only mandatory when using a _UT.Container. Shoudl be set to "ga"

type: "ga",
info

This property is not mandatory but useful in keeping track of Settings versions.

{
date:    "2012-05-12",
author: "DD"
},
args

Object of properties representing all arguments that will be attached to the data object. Each property is a two item Array where the FIRST item is default value and the SECOND is a list of accepted variable names.

args: {
   sPN: ["no_pagename", ["s_pageName", "sPN"]],
   sPath: ["", ["s_path"]]
},

Account used for PRODUCTION data

prod_account: "UA-12345678-1",

For more information, view _setAccount

Comma-delimited list of Accounts that are PRODUCTION

API

The gaAdapter also exposes some methods to simplify the interaction with the _gaq object.

Methods

These methods are available from the this object within the map function.

Method Description & Parameters
fire

Mandatory function inherited from the _UT.Adapter Interface.

Fires both the _trackPageview and _trackTrans if available. Will need to block the trackPageview in certain cases using blockTrackPageView

For more information, view _UT.Adapter.fire
trackEvent

Constructs and sends the event tracking call to the Google Analytics Tracking Code.

// in map()
this.trackEvent(category, action, opt_label, opt_value, opt_noninteraction);

For more information, view _trackEvent
setCustomVar

Sets a custom variable with the supplied name, value, and scope for the variable.

// in map()
this.setCustomVar(index, name, value, opt_scope);

For more information, view _setCustomVar
addTransaction

Creates a transaction object with the given values and adds items to that transaction

  1. trans An object contains transaction information using the following properties:
    • order_id order ID - required
    • store_name affiliation or store name
    • total total - required
    • tax tax
    • shipping shipping
    • city city
    • province state or province
    • country country
  2. prods Array of items to be added to transaction. Each item is an object with the following properties:
    • order_id order ID - required
    • sku SKU/code - required
    • name product name
    • variation category or variation
    • price unit price - required
    • quantity quantity - required
// in map()
this.addTransaction({
	order_id: '1234',		// order ID - required
	store_name: 'Womens Apparel',	// affiliation or store name
	total: '28.28',			// total - required
	tax: '1.29',			// tax
	shipping: '15.00',		// shipping
	city: 'San Jose',			// city
	province: 'California',		// state or province
	country: 'USA'		// country
},[{
	order_id: '1234', 	// order ID - required
	sku: 'DD44',		// SKU/code - required
	name: 'T-Shirt',		// product name
	variation: 'Olive Medium',	// category or variation
	price: '11.99',		// unit price - required
	quantity: '1'	// quantity - required
},{
	order_id: '1235', 	// order ID - required
	sku: 'DD45',		// SKU/code - required
	name: 'Sweat-Shirt',		// product name
	variation: 'Black',	// category or variation
	price: '22.99',		// unit price - required
	quantity: '2'	// quantity - required
}]);

For more information, view Tracking Code: Ecommerce
blockTrackPageView

In case you do want the fire to also call _trackPageview, then use the blockTrackPageView method to disable it.

Note: The trackPageview functionality will be unblocked at the end of the fire action.
// in map()
this.trackEvent(category, action, opt_label, opt_value, opt_noninteraction);
this.blockTrackPageView();
setPageName

Enables you to set the optional pagename URL in _trackPageview.

Note: The trackPageview functionality will be unblocked at the end of the fire action.
// in map()
if(data.is404ServerError) {
	this.setPageName("/404/" + data.sURL );
}

For more information, view _trackPageview
getTrackerName

Retrieves the tracker object with the given name.

Note: By default the name of the tracker is set to settings.id.
// in map()
this.getTrackerName();

For more information, view _getTrackerByName
getTrackerAccount

Returns the Google Analytics ID for this tracker object.

// in map()
this.getTrackerAccount();

For more information, view _getAccount
getTrackerVersion

Returns the GATC version number.

// in map()
this.getTrackerVersion();

For more information, view _getVersion
getVisitorCustomVar

Returns the visitor level custom variable value assigned for the specified index.

// in map()
this.getVisitorCustomVar(1);

For more information, view _getVisitorCustomVar

API

The omniAdapter also exposes some methods to simplify the interaction with the s object.

Methods

These methods are available from the this object within the map function.

Method Description & Parameters
fire

Mandatory function inherited from the _UT.Adapter Interface.

Fires both the _trackPageview and _trackTrans if available. Will need to block the trackPageview in certain cases using blockTrackPageView

For more information, view _UT.Adapter.fire
fireQueueAdd

Function to add args in queue to be fired.

// in map()
this.fireQueueAdd({
   __usePrevious: false,
   __useWindow: true,
   s_pageName: 'pagename:index'
});
clean

Function to empty all s object variables (eVars,props,product,events...).

// in map()
this.clean();
clean

Function to empty all s object variables (eVars,props,product,events...).

// in map()
this.clean();
setTrackLink

Creates a transaction object with the given values and adds items to that transaction

  1. trans An object contains transaction information using the following properties:
    • order_id order ID - required
    • store_name affiliation or store name
    • total total - required
    • tax tax
    • shipping shipping
    • city city
    • province state or province
    • country country
  2. Sets the current beacon to a Custom-Link and attaches variables through the {object} properties being sent:
    1. name Name of custom link
    2. linkTrackEvents Extra events (not counting the settings default) to send
    3. linkTrackVars Extra variables (not counting the settings default) to send
// in map()
this.setTrackLink({
	  name: 'pagename:ajax',
	  linkTrackEvents: 'event32,event33,event34,event35',
	  linkTrackVars: 'eVar31,prop32'
});
addEvent

Method to easily add an event to the events variable. Takes 2 parameters:

  • ev Event in question
  • serializer Optional Serializer
Note: Event adding can be chained.
// in map()
this.addEvent("event21", '123456789').addEvent("event22");
removeEvent

Method to easily remove an event from the events variable. Takes 1 parameter:

  • ev Event in question
Note: Event removal can be chained.
// in map()
this.removeEvent("event21").removeEvent("event22");
addProduct

Method to add products to the products string in an easy manner. Only the first parameter is required:

  • prod {array} of six items representing in order: category, name, quantity, type, incrementors and merchandising
  • inc_merc_1 {object} representing extra merchandising or incrementors (view example)
  • inc_merc_N {object} you can add an unlimited number of these extra parameters
// in map()
this.addProduct(['', 't-shirt', 5, 50, 'event5=3434'], {
    type: 'incrementor',
    value: ['event4=2', 5345345]
}, {
    type: 'merchandising',
    value: 'evar60=cart'
})
purchaseProduct

Method to automatically add a product and purchase success event

// in map()
this.purchaseProduct(['', 't-shirt', 5, 50, 'event5=3434'], {
    type: 'incrementor',
    value: ['event4=2', 5345345]
}, {
    type: 'merchandising',
    value: 'evar60=cart'
})
addEventIncrementor

Method to add an event incrementor with NO product.

  1. ev {string} The event
  2. qty {internger} Incrementor
// in map()
this.addEventIncrementor('event2',10);
cleanVariables

Method to clean and remove empty references form dynamic variables. Takes 2 parameters:

  1. type {string} type of variable to remove: eVar or prop
  2. max_num {number} size of given variable space within the s object (usually 75)
// in map()
this.cleanVariables('eVar',75);
